# 图片剪贴板 — 手机端（HarmonyOS）开发计划

## 当前状态

HarmonyOS 端仅实现了文本剪贴板同步。图片方面：

- **协议层**：`MessageType.CLIPBOARD_IMAGE` 已定义，`SyncMessage` 有 `mimeType` 字段
- **接收**：`SyncManager.ets` 收到 `CLIPBOARD_IMAGE` 时只记录 `'[图片]'` 到历史，**不写入系统剪贴板**
- **发送**：`extractText()` 只处理文本，**不读取剪贴板图片**
- **文件**：完全未实现

---

### 1.2 发送图片（`SyncManager.ets`）

**步骤 1**：新增 `extractImage()` 方法

```typescript
// 在 onClipboardChanged() 中，调用 getData() 后：
// 1. 尝试 extractText() → 有文本则发送文本
// 2. 尝试 extractImage() → 有图片则发送图片

private async extractImage(data: pasteboard.PasteData): Promise<ImagePayload | null> {
  // 获取 PixelMap
  const pixelMap = data.getPrimaryImage()  // 或 data.getImage()
  if (!pixelMap) return null

  // 获取图片信息
  const imageInfo = await pixelMap.getImageInfo()
  const width = imageInfo.size.width
  const height = imageInfo.size.height

  // 计算缩放（长边 > 1920px）
  const maxLongSide = 1920
  let targetWidth = width
  let targetHeight = height
  if (Math.max(width, height) > maxLongSide) {
    const scale = maxLongSide / Math.max(width, height)
    targetWidth = Math.round(width * scale)
    targetHeight = Math.round(height * scale)
  }

  // JPEG 编码（quality=80）
  const packer = image.createImagePacker()
  const opts: image.PackingOption = {
    format: "image/jpeg",
    quality: 80
  }
  const buf = await packer.packing(pixelMap, opts)  // ArrayBuffer
  const base64 = base64Helper.encodeToStringSync(new Uint8Array(buf))

  // 大小检查
  if (buf.byteLength > 20 * 1024 * 1024) {  // 20MB 上限
    console.warn('[SyncManager] Image too large, skipping')
    return null
  }

  return { base64, width: targetWidth, height: targetHeight, size: buf.byteLength, format: 'jpeg' }
}
```

### 1.3 接收图片（`SyncManager.ets`）

修改 `handleRemoteMessage()` 中 `CLIPBOARD_IMAGE` 分支：

```typescript
case MessageType.CLIPBOARD_IMAGE:
  await this.writeClipboardImage(msg)
  this.addRecord('[图片]', 'received')
  break

private async writeClipboardImage(msg: SyncMessage): Promise<void> {
  // 如果有分片 → 进入缓冲逻辑（Phase 2）
  // 单条消息：直接解码写入

  this.isRemoteUpdate = true

  // base64 → ArrayBuffer
  const base64 = msg.content
  const buffer = base64Helper.decodeSync(base64)

  // ArrayBuffer → PixelMap（通过 imageSource）
  const imageSource = image.createImageSource(buffer)
  const pixelMap = await imageSource.createPixelMap()

  // PixelMap → PasteData
  const pasteData = pasteboard.createData(pasteboard.MIMETYPE_IMAGE_PNG, pixelMap)
  // 或者用 MIMETYPE_IMAGE_JPEG 取决于 msg.format

  // 写入系统剪贴板
  systemPasteboard.setDataSync(pasteData)

  // 更新状态
  this.lastChangeCount = systemPasteboard.getChangeCount()
  this.isRemoteUpdate = false
}
```

> **注意**：HarmonyOS `pasteboard` 的图片 MIME 类型常量需要确认。API 版本可能用 `pasteboard.MIMETYPE_IMAGE_PNG` 或 `pasteboard.MIMETYPE_TEXT_PNG`，需查 API 文档。

### 2.2 分片发送（`SyncManager.ets`）

```typescript
private sendInChunks(basePayload: SyncMessage, data: Uint8Array) {
  const CHUNK_SIZE = 256 * 1024  // 256KB
  const totalChunks = Math.ceil(data.byteLength / CHUNK_SIZE)
  const transferId = util.generateRandomUUID()

  for (let i = 0; i < totalChunks; i++) {
    const chunk = data.slice(i * CHUNK_SIZE, Math.min((i + 1) * CHUNK_SIZE, data.byteLength))
    const base64 = base64Helper.encodeToStringSync(chunk)

    if (i === 0) {
      // 首片用原始 type（clipboardImage/clipboardFile），带完整元数据
      const msg = { ...basePayload, content: base64, transferId, chunkIndex: 0, totalChunks }
      this.sendOrBroadcast(msg)
    } else {
      // 后续用 clipboardDataChunk
      const msg: SyncMessage = {
        type: MessageType.CLIPBOARD_DATA_CHUNK,
        content: base64,
        timestamp: Date.now() / 1000,
        deviceId: ProtocolConst.DEVICE_ID,
        transferId,
        chunkIndex: i,
        totalChunks
      }
      this.sendOrBroadcast(msg)
    }
  }
}
```

### 2.3 分片接收缓冲（`SyncManager.ets`）

```typescript
interface ChunkBuffer {
  transferId: string
  totalChunks: number
  chunks: Map<number, string>  // chunkIndex → base64
  metadata: SyncMessage         // 首片消息（含 type, mimeType, fileName 等）
  timestamp: number
}

private chunkBuffers: Map<string, ChunkBuffer> = new Map()

private handleChunk(msg: SyncMessage): boolean {
  // 单条消息（无 transferId 或 totalChunks ≤ 1）→ 直接处理
  if (!msg.transferId || (msg.totalChunks ?? 1) <= 1) return false

  const buffer = this.chunkBuffers.get(msg.transferId) || {
    transferId: msg.transferId,
    totalChunks: msg.totalChunks!,
    chunks: new Map(),
    metadata: msg,  // 首片
    timestamp: Date.now()
  }

  buffer.chunks.set(msg.chunkIndex!, msg.content)

  if (buffer.chunks.size === buffer.totalChunks) {
    // 组装
    let fullBase64 = ''
    for (let i = 0; i < buffer.totalChunks; i++) {
      fullBase64 += buffer.chunks.get(i)
    }
    this.chunkBuffers.delete(msg.transferId)

    // 根据原始 type 处理完整数据
    const fullMsg = { ...buffer.metadata, content: fullBase64 }
    if (buffer.metadata.type === MessageType.CLIPBOARD_IMAGE) {
      this.writeClipboardImage(fullMsg)
    }
    return true
  }

  this.chunkBuffers.set(msg.transferId, buffer)
  return true
}

// 清理：每 30 秒检查超时的缓冲
private startChunkCleanup() {
  this.chunkCleanupTimer = setInterval(() => {
    const now = Date.now()
    for (const [id, buf] of this.chunkBuffers) {
      if (now - buf.timestamp > 30000) {
        console.warn(`[SyncManager] Chunk transfer ${id} timed out`)
        this.chunkBuffers.delete(id)
      }
    }
  }, 30000)
}
```

private async writeClipboardFile(msg: SyncMessage): Promise<void> {
  const base64 = msg.content
  const buffer = base64Helper.decodeSync(base64)

  // 使用 SaveDirectoryManager 获取当前保存目录

## Phase 4：保存目录管理

### 4.1 需求

接收到图片或文件后，保存到用户指定的目录。支持：
- 记住上次使用的目录（持久化到 Preferences）
- 用户可在设置中选择保存目录
- 可将当前目录设为默认

### 4.2 目录选择策略

HarmonyOS 使用 `@kit.CoreFileKit` 的 `FilePicker` / `DocumentViewPicker` 让用户选择目录：

```typescript
// 方案 A：使用 DocumentViewPicker 选择目录
import { picker } from '@kit.CoreFileKit'

async function selectSaveDirectory(): Promise<string> {
  const documentPicker = new picker.DocumentViewPicker()
  const result = await documentPicker.select({
    maxSelectNumber: 1,
    defaultFilePathUri: '/storage/media/100/local/files/Download/'
    // 或使用 fileAccess 权限后的自定义路径
  })
  // result[0] 返回 URI
  return result[0]
}
```

### 4.3 新增 `SaveDirectoryManager`（`model/SaveDirectoryManager.ets`）

```typescript
import { preferences } from '@kit.ArkData'

export class SaveDirectoryManager {
  private static instance: SaveDirectoryManager
  private static readonly KEY_SAVE_DIR = 'save_directory_uri'
  private static readonly DEFAULT_DIR = '/storage/media/100/local/files/Download/ClipboardSync'

  static getInstance(): SaveDirectoryManager {
    if (!SaveDirectoryManager.instance) {
      SaveDirectoryManager.instance = new SaveDirectoryManager()
    }
    return SaveDirectoryManager.instance
  }

  // 获取当前保存目录（记忆的上次目录 或 默认目录）
  getCurrentDirectory(): string {
    const prefs = preferences.getPreferencesSync('clipboard_sync_settings')
    const saved = prefs.getSync(SaveDirectoryManager.KEY_SAVE_DIR, '') as string
    return saved || SaveDirectoryManager.DEFAULT_DIR
  }

  // 设置并记住保存目录
  setDirectory(uri: string): void {
    const prefs = preferences.getPreferencesSync('clipboard_sync_settings')
    prefs.putSync(SaveDirectoryManager.KEY_SAVE_DIR, uri)
    prefs.flushSync()
  }

  // 设为默认（同 setDirectory，语义明确）
  setAsDefault(uri: string): void {
    this.setDirectory(uri)
  }

  // 恢复默认
  resetToDefault(): void {
    const prefs = preferences.getPreferencesSync('clipboard_sync_settings')
    prefs.deleteSync(SaveDirectoryManager.KEY_SAVE_DIR)
    prefs.flushSync()
  }
}
```

### 4.4 UI 入口（`Index.ets`）

在主界面增加「保存位置」设置项：

```typescript
@Component
struct SaveDirectorySetting {
  @State saveDir: string = ''

  aboutToAppear() {
    this.saveDir = SaveDirectoryManager.getInstance().getCurrentDirectory()
  }

  build() {
    Column() {
      Row() {
        Text('保存位置')
        Text(this.saveDir).fontSize(12).maxLines(1)
      }
      .onClick(async () => {
        const dir = await selectSaveDirectory()
        if (dir) {
          SaveDirectoryManager.getInstance().setDirectory(dir)
          this.saveDir = dir
        }
      })

      Button('恢复默认目录')
        .onClick(() => {
          SaveDirectoryManager.getInstance().resetToDefault()
          this.saveDir = SaveDirectoryManager.getInstance().getCurrentDirectory()
        })
    }
  }
}
```

## 验证方式

1. **类型检查**：编译鸿蒙端确保无类型错误
2. **图片发送**：手机复制图片 → 检查日志确认 `extractImage()` 返回有效 base64
3. **图片接收**：Mac 发送 clipboardImage → 手机日志确认图片成功写入剪贴板
4. **分片传输**：发送 >500KB 图片 → 确认分片组装正确
5. **端到端**：两端互发图片，分别在对方设备粘贴验证
6. **保存目录**：选择自定义目录 → 接收文件 → 确认文件保存在所选目录 → 重启应用 → 确认目录记忆生效
7. **默认恢复**：点击恢复默认 → 确认文件再次保存到 `Downloads/ClipboardSync/`
