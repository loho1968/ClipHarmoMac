# 二维码配对计划 —— Mac 生成二维码，鸿蒙扫码自动配对

> **状态**: 规划中  
> **优先级**: P2  
> **目标**: Mac 端展示配对二维码，鸿蒙端扫码自动填入配对码/服务器地址/IP，替代手工输入  
> **预估工期**: 2 天

---

## 1. 当前痛点

目前鸿蒙端所有配对数据都靠手工输入：

| 操作 | 输入内容 | 体验 |
|------|------|:---:|
| 云中继配对 | 6 位 Room Key（Mac 已显示） | 需对照输入 |
| 中继服务器地址 | IP 或域名 | 手工打字 |
| 手动 IP 连接 | Mac 局域网 IP | 手工打字 |

**目标**: Mac 将这些信息合成一个二维码，鸿蒙端扫码一键完成。

---

## 2. 二维码数据格式

### 2.1 JSON 格式（简洁通用）

```json
{
  "v": 1,
  "rk": "A1B2C3",
  "rh": "123.456.12.45",
  "ip": "192.168.1.5"
}
```

| 字段 | 含义 | 必填 | 说明 |
|------|------|:---:|------|
| `v` | 协议版本 | ✅ | 固定为 1，用于将来扩展 |
| `rk` | Room Key（配对码） | ✅ | 6 位大写字母数字 |
| `rh` | Relay Host（中继服务器地址） | ✅ | IP 或域名 |
| `ip` | Mac 局域网 IP | 可选 | 同一 WiFi 下可直连 |

### 2.2 鸿蒙端解析后自动执行

```
扫码 → 解析 JSON
       │
       ├─ rk 存在且 rh 存在 → 自动调用 setRoomKey(rk) + updateRelayHost(rh)
       │                       → 连接中继
       │
       └─ ip 存在            → 可选：自动填入手动连接 IP 输入框
```

## 3. Mac 端 —— 二维码生成

### 3.1 新增文件: `QRCodeGenerator.swift`

```swift
import CoreImage
import AppKit

struct QRCodeGenerator {
    /// 从字符串生成二维码 NSImage
    static func generate(from string: String, size: CGFloat = 200) -> NSImage? {
        let data = string.data(using: .utf8)!
        
        guard let filter = CIFilter(name: "CIQRCodeGenerator") else { return nil }
        filter.setValue(data, forKey: "inputMessage")
        filter.setValue("M", forKey: "inputCorrectionLevel") // M 级纠错
        
        guard let ciImage = filter.outputImage else { return nil }
        
        // 放大到目标尺寸（QR 码像素很小，需要 scale）
        let scaleX = size / ciImage.extent.width
        let scaleY = size / ciImage.extent.height
        let transformed = ciImage.transformed(by: CGAffineTransform(scaleX: scaleX, y: scaleY))
        
        let rep = NSCIImageRep(ciImage: transformed)
        let nsImage = NSImage(size: NSSize(width: size, height: size))
        nsImage.addRepresentation(rep)
        return nsImage
    }
}
```

### 3.2 修改 `SyncManager.swift` —— 新增二维码数据生成

```swift
/// 生成二维码 JSON 字符串
func qrCodeData() -> String {
    var dict: [String: Any] = [
        "v": 1,
        "rk": roomKey,
        "rh": RelayConfig.currentHost,
    ]
    // 附上局域网 IP（手机在同一 WiFi 下可直连）
    if let localIP = getLocalIPAddress() {
        dict["ip"] = localIP
    }
    guard let jsonData = try? JSONSerialization.data(withJSONObject: dict),
          let jsonString = String(data: jsonData, encoding: .utf8) else {
        return ""
    }
    return jsonString
}
```

### 4.2 实现方案：借助系统扫码 Ability

HarmonyOS NEXT 最简单的方式是通过 `startAbilityForResult` 调用系统扫码页：

```typescript
import { common, Want } from '@kit.AbilityKit'

async function startScan(context: common.UIAbilityContext): Promise<string> {
  const want: Want = {
    bundleName: 'com.huawei.hms.scankit',  // 系统扫码组件
    abilityName: 'ScanDefaultAbility',
    parameters: {
      scanFormat: 'ALL',     // 支持所有条码格式
      scanTitle: '扫描配对二维码',
      scanTip: '请将二维码对准取景框',
      enableReturnResult: true
    }
  }
  const result = await context.startAbilityForResult(want)
  return result.resultCode === 0 ? result.want?.parameters?.scanResult as string : ''
}
```

> **备选方案**: 如果系统扫码组件不可用，则使用 `@kit.ScanKit` 内置 `ScanComponent`，更可控但需要更多 UI 代码。

### 4.4 修改 `SyncManager.ets` —— 新增二维码解析方法

```typescript
/**
 * 解析扫码结果并自动配对
 */
processQRCode(qrData: string): void {
  try {
    const data = JSON.parse(qrData) as Record<string, Object>
    if (!data || data.v !== 1) {
      console.error('[SyncManager] Invalid QR code version')
      return
    }
    
    const roomKey = data.rk as string
    const relayHost = data.rh as string
    const localIP = data.ip as string | undefined
    
    if (roomKey && roomKey.length === RelayConfig.ROOM_KEY_LENGTH) {
      // 更新中继服务器地址
      if (relayHost && relayHost.length > 0) {
        this.updateRelayHost(relayHost)
      }
      // 设置配对码并连接中继
      this.setRoomKey(roomKey)
    }
    
    // 同一 WiFi 下也可尝试直连
    if (localIP && localIP.length > 0) {
      this.connectManual(localIP)
    }
  } catch (e) {
    console.error('[SyncManager] QR code parse error: ' + JSON.stringify(e))
  }
}
```
