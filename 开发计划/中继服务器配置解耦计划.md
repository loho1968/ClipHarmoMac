# 中继服务器配置解耦计划 —— 移除硬编码，支持 GitHub 公开

> **状态**: 规划中  
> **优先级**: P1（开源前必须完成）  
> **问题**: 中继服务器 IP `123.456.12.45` 和端口 `8443` 硬编码在源码中，提交到 GitHub 会暴露服务器地址  
> **预估工期**: 1 天

## 2. 当前运行时配置流程

```
目前的工作方式（已有运行时配置能力）：
═══════════════════════════════════════════════════════

Mac 端:
  currentHost 读取链路:
    sharedDefaults.string(forKey: hostDefaultsKey) ?? defaultHost
    └─ 用户修改过？ → 返回已保存的值
    └─ 没修改过？   → 返回硬编码的 "123.456.12.45"

鸿蒙端:
  relayServerHost 读取链路:
    prefs.getSync(RelayConfig.HOST_STORAGE_KEY) ?? RelayConfig.DEFAULT_HOST
    └─ 用户修改过？ → 返回已保存的值
    └─ 没修改过？   → 返回硬编码的 "123.456.12.45"
```

**关键发现**: 运行时已经有完整的持久化链路，用户可以在 UI 中随时修改。问题只在于 **默认值** 被硬编码。

## 4. 各端实现方案

### 4.1 Mac 端 (Swift)

**修改 `Protocol.swift` → `RelayConfig`：**

```swift
enum RelayConfig {
    // ... 保持不变的常量 ...
    static let wsPath = "/ws"
    static let heartbeatInterval: TimeInterval = 30
    static let reconnectBaseDelay: TimeInterval = 1
    static let reconnectMaxDelay: TimeInterval = 30
    static let roomKeyDefaultsKey = "com.clipboardsync.roomKey"
    static let hostDefaultsKey = "com.clipboardsync.relayHost"
    static let roomKeyLength = 6
    static let sharedDefaults = UserDefaults(suiteName: "com.clipboardsync.shared")!

    // 🆕 从配置文件加载默认值
    private static var configDefaults: (host: String, port: Int) = loadConfig()

    static var defaultHost: String { configDefaults.host }
    static var defaultPort: Int { configDefaults.port }

    /// 从 bundle 中的 relay_config.json 读取默认配置
    private static func loadConfig() -> (host: String, port: Int) {
        guard let url = Bundle.main.url(forResource: "relay_config", withExtension: "json"),
              let data = try? Data(contentsOf: url),
              let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any],
              let relay = json["relay"] as? [String: Any] else {
            // 配置文件不存在或格式错误 → 回退到 localhost
            print("[RelayConfig] config file not found, using localhost defaults")
            return ("localhost", 8443)
        }
        let host = relay["defaultHost"] as? String ?? "localhost"
        let port = relay["defaultPort"] as? Int ?? 8443
        return (host, port)
    }

    static var currentHost: String {
        get { sharedDefaults.string(forKey: hostDefaultsKey) ?? defaultHost }
        set { sharedDefaults.set(newValue, forKey: hostDefaultsKey) }
    }

    static var serverURL: URL {
        URL(string: "ws://\(currentHost):\(defaultPort)\(wsPath)")!
    }
}
```

### 4.2 鸿蒙端 (ArkTS)

**当前 `rawfile/` 目录已在 `.gitignore` 中** ✓

**修改 `Protocol.ets` → `RelayConfig`：**

```typescript
import { resourceManager } from '@kit.LocalizationKit'

export class RelayConfig {
  // ... 保持不变的常量 ...
  static readonly WS_PATH: string = '/ws'
  // ...

  // 🆕 懒加载的默认配置
  private static _defaultHost: string = ''
  private static _defaultPort: number = 8443
  private static _configLoaded: boolean = false

  /**
   * 从 rawfile/relay_config.json 加载默认配置
   * 需要在 app 启动时调用一次（传入 Context）
   */
  static async loadConfig(context: Context): Promise<void> {
    if (RelayConfig._configLoaded) return
    
    try {
      const raw = await context.resourceManager.getRawFileContent('relay_config.json')
      const decoder = new util.TextDecoder('utf-8')
      const text = decoder.decode(raw)
      const config = JSON.parse(text)
      if (config.relay) {
        RelayConfig._defaultHost = config.relay.defaultHost || 'localhost'
        RelayConfig._defaultPort = config.relay.defaultPort || 8443
      }
    } catch (_e) {
      // 配置文件不存在 → 回退
      console.info('[RelayConfig] config file not found, using localhost defaults')
      RelayConfig._defaultHost = 'localhost'
    }
    RelayConfig._configLoaded = true
  }

  static get DEFAULT_HOST(): string {
    return RelayConfig._defaultHost || 'localhost'
  }

  static get DEFAULT_PORT(): number {
    return RelayConfig._defaultPort || 8443
  }

  static buildURL(host: string): string {
    return 'ws://' + host + ':' + RelayConfig.DEFAULT_PORT + RelayConfig.WS_PATH
  }
}
```

**修改 `EntryAbility.ets`**（在 `onCreate` 中加载配置）：

```typescript
onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void {
  console.info('[ClipboardSync] EntryAbility onCreate')
  // 🆕 预加载中继配置
  RelayConfig.loadConfig(this.context)
}
```

**注意**: `rawfile/` 目录已在 `.gitignore` 中，但需要确保模板文件不被忽略。方案：

## 6. 回退策略

当配置文件不存在时（如首次 clone 项目的开发者）：

```
┌─────────────────────────────────────────────────┐
│ 配置文件 relay_config.json 存在？                 │
│   ├─ ✅ 是 → 读取 host/port                      │
│   └─ ❌ 否 → 回退 defaultHost = "localhost"      │
│              defaultPort = 8443                   │
│              UI 中显示 "localhost" 作为服务器地址    │
│              用户可以随时在 UI 中修改并持久化       │
└─────────────────────────────────────────────────┘
```

**用户体验**：
- 首次启动，中继服务器地址显示为 `localhost`
- 中继状态显示"无法连接"（因为 localhost 没有中继服务）
- 用户打开中继卡片 → 点击"修改"→ 输入自己的服务器地址 → 自动连接
- 局域网同步不受任何影响（始终可用）

中继服务器用于跨网络（如手机在外面用 4G/5G 时）同步剪贴板。
