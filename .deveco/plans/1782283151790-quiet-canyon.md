## Goal

1. **SettingPage 模板下载**：提供 7 种 CSV 模板下载（景点/兴趣点/航班/租车/住宿/火车/游轮），通过系统文件选择器保存。
2. **TripMain 导入**：选择 CSV 文件批量导入，含行级数据校验、名称去重更新、变化检测、进度条和错误详情展示。

---

## Scope / Non-goals

**Scope：**
- 7 种 CSV 模板
- 所有模板首列为「类型」，非 SpecialSegment（景点/兴趣点）额外包含「分段」列
- 行级校验：类型缺失/无效、必填字段缺失时跳过该行并收集错误
- 名称全局匹配 → 存在则更新、不存在则新增
- 图片 URL / GPS 无变化跳过处理
- 顶部进度条 + 完成后错误详情列表
- SpecialSegment 类型无需「分段」列，导入后存到实际分段，UI 自动生成虚拟分段

**Non-goals：**
- 不生成 .xlsx
- 不涉及数据导出
- 不修改数据模型

---

## Current State And Constraints

- **MVVM + V2 状态管理**：`@ComponentV2`、`@Local`、`@Param`、`@Event`
- **SpecialSegment 定义**：Flight / Accommodation / Rent / Train / Cruise / Restaurant / Bus / Ferry / ToList / Activity（`SegmentViewModel.SPECIAL_ITEM_TYPES`）
- **已有工具**：`ImageDownloadUtil.downloadImage()`、`site.geocode()`、`DocumentViewPicker`
- **ArkTS 约束**：禁止 any/unknown/as、解构、模板字面量、展开运算符；对象字面量必须具名变量；@Builder 中不能声明变量

### 行级校验规则

每行数据在导入前必须通过以下校验，不通过则**跳过该行并记录错误**，不中断整体导入。

| 校验项 | 规则 | 错误信息示例 |
|--------|------|-------------|
| 类型非空 | 第 1 列「类型」不能为空 | `第3行: "类型"不能为空` |
| 类型有效 | 必须是 7 种已知类型之一 | `第3行: 未知类型"机厂"，有效类型: 景点/兴趣点/航班/租车/住宿/火车/游轮` |
| 名称非空（全部） | 各类型的名称列不能为空 | `第5行: "名称"不能为空` |
| 分段非空（仅景点/兴趣点） | 「分段」列不能为空 | `第5行: "分段"不能为空` |

### 导入完整流程

```
用户点击「导入」
  → picker.select(multi .csv, 过滤 .csv)
  → 显示顶部进度条 (Progress + 文字 "正在处理 x/y")
  → 逐文件处理:
      逐行处理:
        ├─ 校验「类型」列 → 无效则记录错误，跳过该行
        ├─ 校验必填字段 → 缺失则记录错误，跳过该行
        ├─ 分段管理: SpecialSegment 用类型中文名查找/创建分段
        │             非 Special 读「分段」列查找/创建分段
        ├─ 名称匹配: tripId + zhName 查已有 item
        │   ├─ 存在 → 比较变化:
        │   │   ├─ 图片URL 无变化 → 跳过下载
        │   │   ├─ 经纬度 无变化 → 跳过 geocode
        │   │   └─ update()
        │   └─ 不存在 → new → insert()
        └─ 更新进度条 (current++, currentItem=名称)
  → 隐藏进度条
  → 显示结果摘要: 成功 N / 更新 M / 跳过 K / 失败 J
  → 如有错误，显示可展开的错误详情列表
  → refreshWishlistVirtualSegments() + refreshSegmentItemCounts()
```

## Execution Sequence

### Step 1: `TemplateGenerator.ets`

- 定义 `TemplateType` 枚举（7 种）
- 每种类型的元数据：类型中文名、是否 SpecialSegment、表头数组
- `generateCsvContent(type: TemplateType): string` 写入 UTF-8 BOM + 表头 + 示例行

### Step 2: `SettingPage.ets`

- 新增 `settingGroupHeader('数据模板')` section
- 7 个下载行，点击触发 `DocumentViewPicker.save()` 保存 CSV

### Step 3: `CsvParser.ets`

- `parseCsv(content: string): string[][]` — 标准 CSV 解析
- `buildFieldIndex(headers: string[], templateType: TemplateType): Record<string, number>` — 列名→索引映射

### Step 4: `TemplateImporter.ets`

核心接口：

```typescript
interface ImportRowError {
  rowNumber: number;
  field: string;
  message: string;
}

interface ImportProgress {
  current: number;
  total: number;
  currentItem: string;
}

interface ImportResult {
  successCount: number;
  updateCount: number;
  skipCount: number;
  failCount: number;
  errors: ImportRowError[];
}

type OnImportProgress = (progress: ImportProgress) => void;
```

核心方法：
- `importCsvFile(uri, tripId, context, onProgress): Promise<ImportResult>`
- 行级校验方法 `validateRow(row: string[], headers: string[]): ImportRowError | null`
  - 校验「类型」非空且有效
  - 校验名称列非空
  - 校验非 Special 的「分段」列非空
- SpecialSegment 分段：自动用 `SPECIAL_TYPE_LABELS[type]` 作分段名
- 名称全局匹配：`SegmentItemRepository` 查询 tripId + zhName
- 变化检测：对比已有 item 的 pictureUrl 和经纬度
- 每行调用 `onProgress` 回调
- 收集所有 `ImportRowError` 到 `errors[]`

### Step 5: `TripMain.ets` + `TripMainNavBar.ets`

- **TripMainNavBar**：新增导入按钮，`@Event onImport`
- **TripMain**：
  - `@Local importProgress: ImportProgress | null = null`
  - `@Local importResult: ImportResult | null = null`
  - 顶部条件渲染进度条 + 文字
  - `handleImport()`：picker.select → 显示进度条 → 逐文件 `importCsvFile()` → 完成后展示结果
  - 结果展示：成功/更新/跳过/失败计数 + 错误列表（可滚动）
  - 导入完成刷新 UI

### Step 6: 资源 + 编译

- 添加字符串资源
- 编译验证

## Verification

| 验证项 | 方法 |
|--------|------|
| 编译通过 | `devecocli build` 无 ERROR |
| 模板下载 | 逐一下载 7 种模板，检查表头和示例 |
| 正常导入 | 用合法 CSV 导入，验证数据入库 |
| 类型缺失/无效 | CSV 中删除或篡改「类型」列 → 验证报错且该行跳过 |
| 名称缺失 | CSV 中删除名称 → 验证报错且该行跳过 |
| 分段缺失（景点/兴趣点） | CSV 中删除「分段」列 → 验证报错 |
| 去重更新 | 同名导入两次 → 验证更新而非重复创建 |
| 变化检测 | 相同图片 URL / GPS 再次导入 → 验证不重复下载/反查 |
| 进度条 | 导入中观察进度条实时刷新 |
| 错误详情 | 含错误行的 CSV 导入后，验证错误详情列表可查看 |
