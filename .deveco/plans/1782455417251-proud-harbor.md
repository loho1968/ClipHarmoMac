## Goal

Transform the expense summary page from a placeholder card layout into a rich, multi-layer visualization:

```
📌 顶部：行程总览卡（总花费 / 人均 / 天数）
📌 第一视觉区：环形图（费用结构，不含预收款）+ 最大分类突出
📌 分摊费用（仅行程结束后显示，仅显示有补/退的团员）
📌 第二分析区：分类排行条形图（不含预收款）
📌 第三记忆区：高消费Top5（不含预收款）
```

Simultaneously:
- Slim expense categories from 13 to 8
- Rename "预交" → "预收款"
- **预收款不计入任何费用统计**（总费用、人均、图表、排行、Top5均排除）
- Make the split-expense section conditional

---

## Scope / Non-goals

**In scope:**
- ExpenseCategory enum rename + EXPENSE_CATEGORIES array reduction (selector keeps 预收款)
- Exclude `Prepay` category from `calculateTotalExpense()`, `calculatePerPersonExpense()`, all chart stats
- New ViewModel calculation methods (category stats, top-N expenses) — all filter out 预收款
- Canvas ring chart component with color-coded arcs (7 categories displayed)
- Category ranking horizontal bar list (7 categories)
- Top-5 most expensive items list (excludes 预收款)
- Trip overview card (total + per-person + trip days)
- Conditional split-expense display (trip-ended check + non-zero filter)

**Out of scope:**
- Changing the add/edit expense dialogs (category dropdown update handled by constant change)
- Chart interaction (tap-to-filter, drill-down) — static display only in this plan
- Migration of existing data — removed categories remain in DB but are excluded from charts

### Existing code state
- [ExpenseSummaryView.ets](entry/src/main/ets/view/ExpenseSummaryView.ets) currently has: total card, per-friend split list (recently refactored), placeholder chart text
- [ExpenseViewModel.ets](entry/src/main/ets/viewmodel/ExpenseViewModel.ets) has: `calculateTotalExpense()`, `calculatePerPersonExpense()`, `calculateCategoryRatio()`, split methods

### ArkTS / Platform constraints
- `@ComponentV2` only; V1 decorators forbidden
- `@Builder` cannot contain `const`/`let` declarations — sub-components needed for computed values
- Canvas API available via `CanvasRenderingContext2D` — arc, text, fill operations
- Color resources must support both light/dark themes
- No third-party chart library — all rendering is custom ArkUI

### Performance
- Ring chart draws once then only re-draws when `allItems` changes (already `@Trace`)
- Category stats use single-pass computation per render

## Design

### Section 0: Category Rename & Slim-down

| Old | New | Action |
|-----|-----|--------|
| `Prepay = '预交'` | `Prepay = '预收款'` | Rename enum value |
| `PublicTransport, Drinks, Shopping, Other, Attraction` | — | Remove from `EXPENSE_CATEGORIES` array; keep enum members for backward-compat |

`EXPENSE_CATEGORIES` after cleanup: `['预收款', '超市', '油费', '餐饮', '活动', '航班', '租车', '住宿']` (8 items)

预收款保留在分类选择器中供记录，但排除在所有费用统计之外。

### Section 0.5: Exclude 预收款 from All Statistics

Add filter to `calculateTotalExpense()`:

```
if (ext.category === ExpenseCategory.Prepay) { continue; }
```

Same filter applies to:
- `calculatePerPersonExpense()` — uses `calculateTotalExpense()` internally
- `calculateCategoryRatio()` / new `getCategoryStats()` — skip prepay items
- `getTopExpenses()` — skip prepay items
- All chart data (ring chart, bar ranking) — show only 7 consumption categories

**But**: split-expense calculations (`calculateSplitTotal`, `calculatePrepayTotal`, `calculateNetExpense`) keep their existing logic where "预收款" items ARE counted (they represent prepaid money that offsets what a friend owes).

### Section 1: Trip Overview Card

Pure ArkUI layout — no Canvas.

```
┌─────────────────────────────┐
│ 行程总览                     │
│ 总花费：12,345.00           │
│ 人均 ：2,469.00             │
│ 行程天数：7 天              │
└─────────────────────────────┘
```

Data: `calculateTotalExpense()` (now excludes 预收款), `calculatePerPersonExpense(friendList.length)`, `trip.durationDays`.

### Section 2: Ring Chart (Canvas)

New component: `CategoryRingChart.ets` in `component/`.

- 7-color palette (no color for 预收款 since it's excluded)
- Draw arcs proportional to category amounts (7 categories displayed)
- Center hole shows total expense text (excluding 预收款)
- Below ring: **max category** row with name + amount + percentage, in accent color
- Legend below ring: ForEach of 7 categories with colored dot + name + amount + percentage

**Canvas drawing logic:**
- `canvas.arc(centerX, centerY, radius, startAngle, endAngle)` for each slice
- Start from 12 o'clock position (-PI/2)
- Each slice arc length = `(categoryAmount / totalChartAmount) * 2 * PI`
- Fill with category color, optional stroke for separation

### Section 3: Split Expense (Conditional) — placed after ring chart

Existing `SplitExpenseBuilder` modified and repositioned:

- **Show condition**: `dayjs().isSameOrAfter(dayjs(trip.endDate), 'day')`
- **Filter**: only friends where `netExpense !== 0` (有补 or 有退)
- Display format unchanged (name, 分摊, 预交, 补/退)
- When hidden: section not rendered at all

New `@Param trip: TripViewModel` added to `ExpenseSummaryView`.

### Section 4: Category Ranking Bar List (7 categories, 不含预收款)

Pure ArkUI `ForEach` over sorted categories — placed after split expense.

Each row: `[category name] [━━━━━━━━ bar width%] [amount] [percentage%]`

- Bar color matches ring chart color
- Bar width = `percentage%` of container width
- Scrollable if > 7 categories

### Section 5: Top-5 Expenses (不含预收款)

New method: `ExpenseViewModel.getTopExpenses(count: number): TopExpenseItem[]`

Sorted by `item.price` descending, exclude `Prepay` category, take top N.

Each row: `[rank #] [category badge] [item name] [amount]`

Rank 1-3 use gold/silver/bronze colored rank numbers; 4-5 use neutral.
