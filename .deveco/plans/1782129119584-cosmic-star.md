# 修复：bindSheet 拖拽手势冲突

## 问题分析

**现象**：在排序弹窗中拖拽列表项排序时，容易连带拖动整个半模态弹窗向下滑动（触发 sheet 的 swipe-to-dismiss 手势）。

**根因**：`onDragStart` 的长按+拖拽手势与 bindSheet 内置的下滑关闭手势产生竞争。虽然已设置 `dragBar: false` 隐藏了拖拽条，但 sheet 仍通过内容区域的垂直滑动手势支持 dismiss。拖拽排序项的垂直分量被 sheet 解释为下滑手势。

ArkUI 的嵌套滚动默认行为：
```
{Forward: PARENT_FIRST, Backward: SELF_FIRST}
```
- 向前（上滑）：父级（sheet）优先
- 向后（下滑）：自身（List）优先

但拖拽手势（`onDragStart`）不属于滚动体系，不受嵌套滚动优先级控制。

## 解决方案

使用 bindSheet 的 `onWillDismiss` + `onWillSpringBackWhenDismiss` 双保险策略：

### 1. `onWillDismiss` — 拦截下滑关闭

只允许通过代码关闭（关闭按钮/取消按钮），拒绝 `DismissReason.SlideDown`：

```typescript
onWillDismiss: (dismissSheetAction: DismissSheetAction) => {
    if (dismissSheetAction.reason === DismissReason.PRESS_BACK) {
        dismissSheetAction.dismiss();
    }
    // SlideDown 不处理 → 下滑关闭被彻底禁用
}
```

### 2. `onWillSpringBackWhenDismiss` — 阻止回弹动画

即使 sheet 不被关闭，下滑时可能产生弹簧回弹视觉效果。通过不调用 `springBack()` 阻止：

```typescript
onWillSpringBackWhenDismiss: (springBackAction: SpringBackAction) => {
    // 不调用 springBack() → 无回弹动画
}
```

### 3. 已有保障

- `dragBar: false` — 无视觉拖拽条
- 关闭按钮（✕）和"取消"按钮通过 `onClose` → `showSortDialog = false` 关闭
