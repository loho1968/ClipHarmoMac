## 目标

熄屏后再亮屏，**不需要打开 App 界面**，就能自动收到 Mac 端复制的剪贴板。

## 设计

### 核心思路

利用 `MULTI_DEVICE_CONNECTION` 保活进程 + 后台定时器定期拉取。

```
后台定时器（每 30 秒 + power.isActive 亮屏检测）
  → 检查 TCP/WS 连接是否存活
  → 是：发送 CLIPBOARD_POLL
  → 否：跳过（等连接恢复后再触发）
```

### power.isActive() 加速亮屏同步

`power.isActive()` 检测设备是否处于活动状态（亮屏 = true，熄屏 = false）。在定时器中监控该值的变化，从 false→true 时立即发送 POLL，避免等满 30 秒。

### 关键文件变更

#### `[SyncManager.ets](entry/src/main/ets/model/SyncManager.ets)`

**新增成员：**
```typescript
private backgroundPollTimerId: number = -1
private screenWasOff: boolean = false
```

**`start()` 末尾追加：**
```typescript
this.startBackgroundPoll()
```

**`stop()` 中追加：**
```typescript
this.stopBackgroundPoll()
```

**新增方法：**
```typescript
private startBackgroundPoll(): void {
  this.backgroundPollTimerId = setInterval(() => {
    // 亮屏检测：之前熄屏 → 现在亮屏 → 立即 POLL
    try {
      const active = power.isActive()
      if (active && this.screenWasOff) {
        console.info('[SyncManager] screen woke up, polling immediately')
        this.requestRemoteClipboard()
      }
      this.screenWasOff = !active
    } catch (_e) { /* API not available */ }

    // 定期 POLL（30 秒）
    if (this.tcpClient.connected || this.wsClient.connected) {
      this.requestRemoteClipboard()
    }
  }, 30000) as number
  console.info('[SyncManager] background poll started (30s interval)')
}

private stopBackgroundPoll(): void {
  if (this.backgroundPollTimerId !== -1) {
    clearInterval(this.backgroundPollTimerId)
    this.backgroundPollTimerId = -1
  }
}
```

**导入：**
```typescript
import { power } from '@kit.BasicServicesKit'
```

### 触发时机总结

| 触发 | 间隔 | 场景 |
|------|------|------|
| 后台定时器 | 每 30s | 熄屏期间持续拉取 |
| power.isActive 亮屏检测 | 瞬间 | 熄屏→亮屏瞬间加速 |
| onForeground | 打开 App | 兜底触发 |
| TCP/WS 连接建立 | 连接时 | 断连重连后触发 |

## 验证

| 检查项 | 方法 |
|--------|------|
| 熄屏 1 分钟后亮屏 | Mac 复制 → 手机亮屏 → 30s 内打开任意 App 粘贴 → 已有文字 |
| 不打开 App 亮屏 | 熄屏 → Mac 复制 → 亮屏（不打开 ClipboardSync）→ 打开微信粘贴 → 已有文字 |
| 后台定时器功耗 | 30s × 1 条 <100 字节消息 = 极低耗电 |

## 回滚

删除 `startBackgroundPoll`/`stopBackgroundPoll` 及相关调用即可。
