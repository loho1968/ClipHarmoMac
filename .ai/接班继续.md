# 接班继续指令模版

> 换机器后（如从公司回家），在新机器上执行此指令，从交接文件恢复工作上下文。

---

## 前提条件

1. 项目代码已在新机器上（`git clone` 或已有仓库）
2. `.ai/HANDOFF.md` 已存在于仓库中（通过 git 同步或手动拷贝）
3. 新机器的开发环境已就绪（Xcode、DevEco Studio、Node.js 等）

---

## 使用方式

将下方指令块复制给 AI 助手执行。

---

## 指令

```
我换机器了，请从交接文件恢复工作上下文。依次完成：

---

### 第 1 步：读取交接文件

读取 `.ai/HANDOFF.md`，展示关键信息摘要：

- 日期
- 原机器主机名
- 原分支和最后提交
- 今日已完成事项
- 工作断点

---

### 第 2 步：检查新机器环境

运行以下命令，对照交接文件检查差异：

1. **同步代码**
   ```bash
   git fetch origin
   git log --oneline -5
   ```
   确认本地是否已拉取到交接文件中的最新提交。

2. **检查分支**
   ```bash
   git branch --show-current
   git status
   ```
   如果当前分支与交接文件不一致，提醒切换到对应分支。

3. **检查未同步的提交**
   ```bash
   git log origin/$(git branch --show-current)..HEAD --oneline
   ```
   如果有本地独有提交，提醒 push。

4. **构建验证**
   ```bash
   cd ClipboardSync/mac && swift build 2>&1 | tail -10
   ```

5. **配置文件检查**
   ```bash
   # Mac 端中继配置
   cat ~/.clipboardsync/relay_config.json 2>/dev/null || echo "⚠️ 配置文件不存在，需要创建"
   
   # Homebrew 依赖（如有）
   which swift 2>/dev/null || echo "⚠️ Swift 工具链未找到"
   ```

---

### 第 3 步：环境差异报告

对比原机器和新机器，列出差异：

| 检查项 | 原机器 | 新机器 | 状态 |
|--------|--------|--------|------|
| 分支 | <交接文件> | <当前> | ✅/⚠️ |
| 提交 | <交接文件> | <当前> | ✅/⚠️ |
| 构建 | <交接文件> | <当前> | ✅/⚠️ |
| 配置文件 | - | 存在/缺失 | ✅/⚠️ |
| 未提交变更 | N 个文件 | - | ℹ️ |

对每个 ⚠️ 项给出修复建议。

---

### 第 4 步：恢复工作上下文

根据交接文件中的「工作断点」和「文件变更清单」：

1. **恢复未提交的变更**（如果有 stash 或暂存的改动未 push）：
   - 检查 `git stash list` 是否为空
   - 提醒用户原机器的未提交变更需要在原机器上 commit + push 才能同步

2. **打开关键文件**：列出断点涉及的核心文件，让用户快速定位

3. **重建心智模型**：用 3-5 句话总结：
   - 项目是做什么的（从 CONTEXT.md 取一句话）
   - 昨天做了什么
   - 今天从哪里开始
   - 关键注意事项

---

### 第 5 步：生成「今天的第一件事」

根据交接文件中的优先级排序，输出今天的工作清单：

```
📋 今天的工作清单：

1. [P0] <第一件事 — 具体可执行的下一步>
2. [P1] <后续事项>
3. [P2] <低优先级>

🔑 关键文件：
- ClipboardSync/mac/ClipboardSync/<文件>.swift — <为什么重要>
- ClipboardSync/harmony/.../<文件>.ets — <为什么重要>
```

---

### 第 6 步：确认就绪

最后输出：

```
✅ 环境就绪，可以开始工作了。

如果遇到编译错误，说「帮我看下编译错误」。
如果要继续开发昨天的功能，说「继续昨天的工作」。
```
```

---

## 场景变体

### 场景 A：同一台机器继续（没换机器）

简化指令：

```
继续昨天的工作。读 .ai/HANDOFF.md，检查 git status，告诉我今天该做什么。
```

### 场景 B：交接文件不在仓库中

如果 `.ai/HANDOFF.md` 因未提交而不同步，需要手动从原机器拷贝：

```bash
# 在原机器上
scp .ai/HANDOFF.md user@home-machine:/path/to/ClipHarmoMac/.ai/
```

### 场景 C：快速恢复（跳过环境检查）

只关注工作断点的最短指令：

```
读 .ai/HANDOFF.md，告诉我上次停在哪里、今天该做什么。
```
