# 下班交接指令模版

> 下班前在**当前机器**上执行此指令，生成交接摘要。
> 回家后打开另一台机器，用「接班继续」指令恢复上下文。

---

## 使用方式

将下方指令块复制给 AI 助手执行，或在终端中依次运行。

---

## 指令

```
请帮我生成下班交接摘要。依次完成以下步骤：

---

### 第 1 步：捕获当前工作状态

运行以下命令，原样输出结果：

1. **Git 状态**
   ```bash
   git status
   git log --oneline -10
   git stash list
   ```

2. **当前分支与未提交变更**（⚠️ 三类都要查：未暂存 + 已暂存 + 未跟踪）
   ```bash
   git branch --show-current
   echo "=== 未暂存的变更（已跟踪文件）==="
   git diff --stat
   echo "=== 已暂存未提交 ==="
   git diff --cached --stat
   echo "=== 未跟踪的新文件（最容易遗漏！）==="
   git ls-files --others --exclude-standard
   echo "=== 已提交但未推送 ==="
   git log origin/$(git branch --show-current)..HEAD --oneline
   ```

3. **项目文件变更概览**（最近一天修改过的文件）
   ```bash
   find . -type f \( -name "*.swift" -o -name "*.ets" -o -name "*.js" -o -name "*.ts" -o -name "*.json" -o -name "*.md" -o -name "*.py" -o -name "*.sh" \) -mtime -1 | grep -v node_modules | grep -v .build | grep -v .git | sort
   ```

4. **构建状态**
   ```bash
   cd ClipboardSync/mac && swift build 2>&1 | tail -5
   ```

---

### 第 2 步：总结今日工作

根据 git diff、git log 和项目文件变更，用中文总结：

1. **今日完成了什么**（按功能点列出，每条一句话）
2. **修改了哪些文件**（文件路径 + 一句话说明改了什么）
3. **发现的问题 / 踩坑记录**（如果有）
4. **当前编译状态**：通过 / 有报错（附错误概要）
5. **关键决策或设计变更**（如果有，一句话）

---

### 第 3 步：标记工作断点

1. **正在进行但未完成的事项**：
   - 当前正在写什么功能？
   - 卡在哪个具体问题上？
   - 下一步打算做什么？

2. **优先级排序**（按 P0 > P1 > P2）：
   - 明天第一件事该做什么？

---

### 第 4 步：生成交接文件

将以上所有信息整理后，**覆盖写入** `.ai/HANDOFF.md`，格式如下：

```markdown
# 工作交接 — 2025-07-XX

## 机器信息
- **主机名**: <从命令获取>
- **分支**: <当前分支>
- **最后提交**: <最新 commit hash + message>

## 今日完成
- [事项 1]
- [事项 2]

## 文件变更清单
| 文件 | 变更说明 |
|------|---------|
| ... | ... |

## 编译状态
<通过 or 报错详情>

## Git 状态
- 未暂存修改: N 个文件
- 已暂存未提交: M 个文件
- 未跟踪新文件: K 个文件
- 未推送提交: L 个
- ⚠️ <如有未提交文件，必须在此列出！>

## 工作断点
- **正在做**: <描述>
- **卡在哪里**: <描述 or 无>
- **下一步**: <描述>

## 关键决策
- <决策描述 or 无>

## 待清理
- [ ] <如有 stash / 临时文件 / 调试代码需清理>
```

---

### 第 5 步：自动提交 & 推送

**将今日所有变更（包括交接文件）自动提交并推送：**

```bash
git add -A
git commit -m "chore: 下班交接 — $(date +%Y-%m-%d)" && git push
```

提交后输出简要确认：

```
✅ 已提交并推送：<commit hash> — chore: 下班交接 — 2025-07-03
```

如果 `git add -A` 后无变更（工作区已干净），则：

```
✅ 工作区已干净，无需提交。
```

如果 push 失败（如无网络），则提醒用户稍后手动 `git push`。

---

### 第 6 步：输出同机器恢复提示

告诉用户：如果在同一台机器上继续，只需说「继续昨天的工作」。

如果换了机器，需说「我换机器了，从交接文件恢复」，然后 AI 会读取 `.ai/HANDOFF.md` 并执行接班流程。
```

---

## 快速版（简化指令）

如果时间紧，直接用这条最短指令：

```
生成下班交接：运行 git status、git log -5、git diff --stat、git ls-files --others --exclude-standard，
总结今日完成事项和未完成断点，写入 .ai/HANDOFF.md。如有未提交文件务必提醒。
```
